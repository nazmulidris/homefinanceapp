Webservice endpoints

  • test1/ webservice endpoint - This simply prints out the values of the params (q, u) that are passed in via the URL
  • test2/ webservice endpoint - This takes the value of the zip param that's passed via the URL and if the value is "94301" or "94040" then it prints out city names, otherwise it just prints out the zip code.
  • weather/ webservice endpoint - This takes the value of the param zip (which is a zip code) and uses the openweathermap.org webservice in order to get the current weather conditions for this zip code. Then it returns a JSON object with the current temperature in Celsius and Fahrenheit, and the name of the city. In case there's a mistake the the zip code format, it returns an JSON object w/ error messages in it. Also, if no zip URL param is passed, then it uses 94301.

Apps

  • weather.html weather app - This uses the weather/ endpoint to create a simple UI that allows the user to get the current temperature for a given zip code.
